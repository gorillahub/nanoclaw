Create `src/session-awareness.ts` with these exports:

```typescript
interface ActiveSession {
  containerId: string;
  started: string;     // ISO 8601
  type: 'message' | 'task';
  repos: string[];     // empty for now — populated in future phases
}

interface ActiveSessionsFile {
  sessions: ActiveSession[];
  updatedAt: string;   // ISO 8601
}
```

**`writeActiveSessionsFile(groupFolder: string, session: ActiveSession): void`**
- Reads existing `data/ipc/{groupFolder}/active_sessions.json` (or starts empty)
- Appends the new session to the `sessions` array
- Writes atomically (temp + rename pattern — already used in group-queue.ts `sendMessage`)
- Sets `updatedAt` to current ISO timestamp
- Uses `resolveGroupIpcPath()` from `group-folder.ts` for path resolution (security)
- Creates the IPC directory if it doesn't exist (`fs.mkdirSync recursive`)

**`removeActiveSession(groupFolder: string, containerId: string): void`**
- Reads existing file, filters out the session with matching `containerId`
- Writes back atomically
- If no sessions remain, writes `{ sessions: [], updatedAt: "..." }` (don't delete file — container might read it)

**`readActiveSessionsFile(groupFolder: string): ActiveSessionsFile`**
- Reads and parses the file, returns `{ sessions: [], updatedAt: "" }` if file doesn't exist or is corrupt
- Export this for container-side to potentially use, but mainly for tests

All file operations are synchronous (Node.js single-threaded, no locking needed — per project constraints).

**GroupQueue changes (`src/group-queue.ts`):**

Add optional callback hooks to GroupQueue for container lifecycle events. Do NOT import session-awareness.ts directly — keep GroupQueue decoupled (it's already callback-driven via `setProcessMessagesFn`).

Add two new optional callbacks:

```typescript
private onContainerStartFn: ((groupFolder: string, session: { containerId: string; type: 'message' | 'task' }) => void) | null = null;
private onContainerExitFn: ((groupFolder: string, containerId: string) => void) | null = null;
```

Add setter methods:
```typescript
setOnContainerStart(fn: (groupFolder: string, session: { containerId: string; type: 'message' | 'task' }) => void): void
setOnContainerExit(fn: (groupFolder: string, containerId: string) => void): void
```

Call `onContainerStartFn` in `runForGroup()` — immediately AFTER `state.containers.set(containerId, slot)` and AFTER the slot has its containerId and type set. The `groupFolder` is not available on the slot at this point (it's set later by `registerProcess`), so the callback must receive the groupFolder. Problem: at `runForGroup` time, we don't know the groupFolder yet — it's set by `registerProcess`.

**Better approach:** Call `onContainerStartFn` inside `registerProcess()` — this is when we know both the containerId AND the groupFolder. Only fire if `groupFolder` is provided (it's optional for backward compat). This is also closer to when the container is actually running (process registered = container spawned).

```typescript
// In registerProcess(), after slot.groupFolder is set:
if (groupFolder && this.onContainerStartFn) {
  this.onContainerStartFn(groupFolder, { containerId: slot.containerId, type: slot.type });
}
```

Call `onContainerExitFn` in the `finally` blocks of BOTH `runForGroup()` and `runTask()` — immediately BEFORE `state.containers.delete(containerId)`. At this point the slot still has its groupFolder. Only fire if `slot.groupFolder` is set.

```typescript
// In runForGroup() finally, before state.containers.delete:
if (slot.groupFolder && this.onContainerExitFn) {
  this.onContainerExitFn(slot.groupFolder, containerId);
}
```

Same pattern in `runTask()` finally block.

**Index.ts changes (`src/index.ts`):**

Import `writeActiveSessionsFile` and `removeActiveSession` from `./session-awareness.js`.

After `const queue = new GroupQueue()` (around line 67), wire the callbacks:

```typescript
queue.setOnContainerStart((groupFolder, session) => {
  writeActiveSessionsFile(groupFolder, {
    containerId: session.containerId,
    started: new Date().toISOString(),
    type: session.type,
    repos: [],
  });
});

queue.setOnContainerExit((groupFolder, containerId) => {
  removeActiveSession(groupFolder, containerId);
});
```

This keeps all session-awareness logic out of GroupQueue itself — clean separation.
