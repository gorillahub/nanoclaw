**Add a helper function `readSessionAwareness()`:**

```typescript
function readSessionAwareness(): string {
  const awarenessPath = '/workspace/ipc/active_sessions.json';
  try {
    if (!fs.existsSync(awarenessPath)) return '';
    const data = JSON.parse(fs.readFileSync(awarenessPath, 'utf-8'));
    const sessions = data.sessions;
    if (!Array.isArray(sessions) || sessions.length === 0) return '';

    const lines = sessions.map((s: { containerId: string; started: string; type: string; repos: string[] }) =>
      `  <session containerId="${s.containerId}" started="${s.started}" type="${s.type}"${s.repos?.length ? ` repos="${s.repos.join(', ')}"` : ''} />`
    );
    return `<active-sessions>\n${lines.join('\n')}\n</active-sessions>`;
  } catch (err) {
    log(`Failed to read session awareness: ${err instanceof Error ? err.message : String(err)}`);
    return '';
  }
}
```

**Inject into the initial prompt:**

In `main()`, after the prompt is assembled (after the scheduled task prefix and pending IPC drain, around line 536), prepend the awareness context:

const awarenessContext = readSessionAwareness();

This goes AFTER the existing `if (containerInput.isScheduledTask)` block and `drainIpcInput()` call, but BEFORE the query loop (`while (true)`).

**Important notes:**
- Read ONCE on startup, not on every query iteration. The awareness snapshot is point-in-time — the agent doesn't need live updates during its run.
- Gracefully handle: file doesn't exist (new group, first container), file is empty JSON, file is corrupt, `sessions` array is empty.
- The `<active-sessions>` XML block matches the pattern Holly/Claude already understands from system prompts — XML context blocks are standard.
- Do NOT add this to follow-up messages (the IPC piped messages). Only the initial prompt needs awareness.
- Use `/workspace/ipc/active_sessions.json` — this is where the host writes it (the IPC directory is already mounted at `/workspace/ipc/`).
